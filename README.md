# cpp-snippets

Collection of C++ examples and exercises organized by topics. Each folder
contains a `.cpp` version and, when applicable, a subdirectory with the same
exercise name.

## Structure

- `1-10/`, `11-20/`, `21-30/`: exercises organized in sequence.
- `hello-world.cpp`: simple "Hello World" example.

## Install tools (Linux)

Below are common ways to install a C++ compiler on Linux.

### Debian/Ubuntu

```bash
sudo apt update
sudo apt install build-essential
```

### Fedora

```bash
sudo dnf install gcc-c++
```

### Arch

```bash
sudo pacman -S base-devel
```

## Build and run

Example using GCC (g++):

```bash
g++ -std=c++17 21-30/26-rock-paper-scissors.cpp -o rps
./rps
```

Replace the `.cpp` path with the exercise you want.

If you want to compile with warnings enabled:

```bash
g++ -std=c++17 -Wall -Wextra -Wpedantic 21-30/26-rock-paper-scissors.cpp -o rps
```

## Requirements

- C++ compiler (e.g., `g++`).

## Notes

- `tempCodeRunnerFile.cpp` is generated by some IDEs and can be ignored.
- Info about the code: ```mkdir -p build_info && g++ -O2 -g main.cpp -o app && objdump -dS app > build_info/disasm_with_source.txt && readelf -SW app > build_info/sections.txt && nm -n app > build_info/symbols.txt
```
- Info with warning: ```mkdir -p build_info && g++ -O2 -g main.cpp -o app 2> build_info/gpp_err.txt && objdump -dS app > build_info/disasm_with_source.txt 2> build_info/objdump_err.txt && readelf -SW app > build_info/sections.txt 2> build_info/readelf_err.txt && nm -n app > build_info/symbols.txt 2> build_info/nm_err.txt
```
